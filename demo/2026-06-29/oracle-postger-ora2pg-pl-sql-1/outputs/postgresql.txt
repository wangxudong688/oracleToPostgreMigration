PostgreSQL 常用语句，按场景整理了一份：
数据库 & 表管理
-- 连接数据库
psql -U username -d dbname -h host -p port

-- 创建/删除数据库
CREATE DATABASE dbname;
DROP DATABASE dbname;

-- 列出所有数据库
\l

-- 切换到数据库
\c dbname

-- 列出所有表
\dt

-- 查看表结构
\d tablename

-- 创建表
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 删除表
DROP TABLE IF EXISTS tablename;

-- 清空表（重置自增序列）
TRUNCATE TABLE tablename RESTART IDENTITY;
增删改查
-- 插入
INSERT INTO users (name, email) VALUES ('Alice', 'alice@example.com');
INSERT INTO users (name, email) VALUES ('Bob', 'bob@example.com'), ('Carol', 'carol@example.com');

-- 查询
SELECT * FROM users;
SELECT id, name FROM users WHERE name LIKE 'A%' ORDER BY created_at DESC LIMIT 10;

-- 更新
UPDATE users SET email = 'new@example.com' WHERE id = 1;

-- 删除
DELETE FROM users WHERE id = 1;
聚合 & 分组
SELECT department, COUNT(*), AVG(salary), MAX(salary), MIN(salary)
FROM employees
GROUP BY department
HAVING COUNT(*) > 5
ORDER BY AVG(salary) DESC;
连接
-- INNER / LEFT / RIGHT / FULL JOIN
SELECT u.name, o.order_id, o.amount
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.amount IS NULL;  -- 没下过单的用户
索引
CREATE INDEX idx_users_email ON users(email);
CREATE UNIQUE INDEX idx_users_email_unique ON users(email);
CREATE INDEX idx_users_name_created ON users(name, created_at);

-- 查看索引
\di

-- 删除索引
DROP INDEX idx_users_email;
进阶查询
-- 分页
SELECT * FROM users ORDER BY id LIMIT 20 OFFSET 40;

-- 去重
SELECT DISTINCT department FROM employees;

-- 子查询
SELECT * FROM users WHERE id IN (SELECT user_id FROM orders WHERE amount > 100);

-- CTE (公用表表达式)
WITH recent_orders AS (
    SELECT user_id, SUM(amount) AS total
    FROM orders
    WHERE created_at >= NOW() - INTERVAL '30 days'
    GROUP BY user_id
)
SELECT u.name, COALESCE(r.total, 0) AS total_30d
FROM users u
LEFT JOIN recent_orders r ON u.id = r.user_id;
窗口函数
SELECT name, department, salary,
       RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rank,
       ROW_NUMBER() OVER (ORDER BY salary DESC) AS row_num,
       LAG(salary) OVER (PARTITION BY department ORDER BY salary) AS prev_salary,
       AVG(salary) OVER (PARTITION BY department) AS dept_avg
FROM employees;
事务
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;
-- 或 ROLLBACK;
常用函数
-- 时间
NOW(), CURRENT_DATE, AGE(timestamp, timestamp),
DATE_TRUNC('month', created_at), EXTRACT(YEAR FROM created_at)

-- 字符串
LENGTH(), UPPER(), LOWER(), TRIM(), CONCAT(a, b),
SUBSTRING('hello' FROM 2 FOR 3), REPLACE(str, old, new)

-- 类型转换
'123'::INTEGER, CAST('2024-01-01' AS DATE)

-- 条件
COALESCE(col, 'default'), NULLIF(a, b),
CASE WHEN condition THEN result ELSE default END

-- JSON / JSONB
data->'key', data->>'key', data#>'{path,to,key}',
jsonb_array_elements(data->'items')
实用管理命令（psql 内）
\dt          -- 列出表
\di          -- 列出索引
\dv          -- 列出视图
\df          -- 列出函数
\x           -- 扩展显示（行转列）
\timing      -- 显示执行时间
\o file.sql  -- 将输出写入文件
\i file.sql  -- 执行 SQL 文件
性能分析
EXPLAIN SELECT * FROM users WHERE email = '...';
EXPLAIN ANALYZE SELECT * FROM users WHERE email = '...';

-- 查看慢查询（需要 pg_stat_statements 扩展）
SELECT query, calls, total_time, rows
FROM pg_stat_statements
ORDER BY total_time DESC LIMIT 10;

在 PostgreSQL 的 psql 命令行工具中，所有以 \ 开头的都是元命令（Meta-Commands），用于查看和管理数据库对象。以下是常用命令总结：

👤 用户与权限管理
命令	作用
\du	列出所有数据库用户（角色）及其属性
\du+	更详细地列出用户信息（包含描述）
\dg	同 \du，列出所有角色
\drds	显示每个数据库的角色设置
\dp 或 \z	显示表的权限分配情况
🗄️ 数据库与表对象查看
命令	作用
\l	列出所有数据库
\l+	更详细地列出数据库（包含大小、描述）
\c <数据库名>	切换到指定数据库
\dt	列出当前数据库中所有表
\dt+	更详细地列出表（包含大小、描述）
\dt *.*	列出所有模式下的所有表
\dt testsc.*	列出 testsc 模式下的所有表
\d	列出当前数据库的所有对象（表、视图、序列等）
\d <表名>	查看指定表的结构（字段、类型、约束）
\d+ <表名>	更详细地查看表结构（包含存储信息）
\di	列出所有索引
\dv	列出所有视图
\ds	列出所有序列
\df	列出所有函数
\df+	更详细地列出函数（包含源代码）
🧩 模式（Schema）管理
命令	作用
\dn	列出所有模式
\dn+	更详细地列出模式（包含描述）
\dx	列出所有已安装的扩展（Extensions）
📊 查询与执行相关
命令	作用
\g	重复执行上一条 SQL 命令
\g <文件名>	将上一条 SQL 的执行结果输出到文件
\x	切换扩展显示模式（按列显示，便于查看宽表）
\x on / \x off	开启/关闭扩展显示模式
\timing	切换 SQL 执行时间显示（显示每条 SQL 耗时）
\o <文件名>	将后续所有查询结果输出到文件
\o	恢复输出到终端
🛠️ 系统信息与配置
命令	作用
\conninfo	显示当前连接信息（数据库、用户、端口）
\encoding	显示当前客户端编码
\version	显示 PostgreSQL 版本信息
\show	显示运行时配置参数（如 \show search_path）
\set	显示或设置 psql 变量
\unset	取消设置 psql 变量
📁 文件与外部操作
命令	作用
\i <文件名>	执行 SQL 文件（导入）
\ir <文件名>	执行 SQL 文件（相对于当前目录）
\o <文件名>	将输出重定向到文件
\copy	执行导入/导出数据操作（比 COPY 更灵活）
\! <系统命令>	执行系统 shell 命令（如 \! ls）
🧹 历史与帮助
命令	作用
\h 或 \help	查看 SQL 命令帮助
\h <SQL命令>	查看特定 SQL 命令的帮助（如 \h SELECT）
\?	列出所有元命令的帮助
\s	显示命令历史
\s <文件名>	将命令历史保存到文件
\q	退出 psql
🎨 显示格式控制
命令	作用
\pset format <格式>	设置输出格式（如 aligned、csv、html、latex 等）
\pset border <值>	设置表格边框样式
\a	切换非对齐模式
\H	切换 HTML 输出模式
\t	切换只显示数据（不显示列名和行数）
💡 实用组合技巧
组合命令	作用
\l+	查看所有数据库及大小
\dt+	查看所有表及大小
\d+ 表名	查看表结构、索引、约束等完整信息
\x \dt	扩展显示模式下查看表列表（便于横向阅读）
\du+	查看用户详细权限信息
📝 总结
这些元命令的核心用途：

\du → 查看用户权限（你问过的）

\dt → 查看表（当 \d 显示 "Did not find any relations" 时，用这个指定查看表）

\dn → 查看模式（确认表在哪个模式下）

\l → 查看数据库（确认当前在哪个库）

\c → 切换数据库（解决连接错库的问题）
