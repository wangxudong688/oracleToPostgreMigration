2. 修改配置文件
编辑配置文件config/ora2pg.conf，设置Oracle数据库连接信息和迁移选项。

3. 执行模式导出
cd /app/migration/oracle_to_pg
./export_schema.sh
4. 检查生成的SQL文件
导出完成后，检查schema目录下生成的SQL文件，特别是存储过程和触发器，可能需要手动调整。

5. 导入到PostgreSQL

直接回答用法。
import_all.sh 必须在 ora2pg 的项目目录（有 config/、schema/、data/ 子目录）下运行。
必选参数是两个：
./import_all.sh -d 数据库名 -o 数据库属主
常用参数：
参数	说明
-d dbname	目标 PostgreSQL 数据库名（必填）
-o owner	数据库属主用户名（必填）
-h host	PostgreSQL 主机地址，默认 unix socket
-p port	PostgreSQL 端口，默认 5432
-U user	连接 PostgreSQL 的用户名
-n schema	要创建的 schema 列表，逗号分隔
-s	只导入 schema，不导入数据
-a	只导入数据，不导入 schema
-y	自动回答 yes，跳过所有确认提示

如果你的表在 public 以外的 schema 里，加上 -n：
./import_all.sh -d mydb -o myuser -n my_schema -y
-y 通常建议加上，否则每一步都要手动确认。


脚本本身有两个方式：
1. 用 -l 参数指定日志文件
./import_all.sh -d mydb -o myuser -y -l /tmp/import.log
所有 stdout 和 stderr 都会写入这个文件，查看错误：
grep -i error /tmp/import.log
2. 或者自己重定向输出
./import_all.sh -d mydb -o myuser -y 2>&1 | tee import_all.log
脚本遇到错误会立即停（die() 函数），并打印 ERROR: 开头的信息到 stderr。搜索关键词 ERROR 或 FATAL 就能定位到失败点。
如果只看最后的错误：
tail -50 import_all.log | grep -i -E 'error|fatal|failure'
另外，脚本执行每个 psql 后都会检查返回值，非零就停。所以如果你看到脚本中途退出，最后一个 psql -f 执行的文件就是出错的地方。




用法
import_all.sh 是 Ora2Pg 生成的导入工具，负责把导出好的 SQL 文件（schema + data）导入 PostgreSQL。基本用法：
./import_all.sh -d dbname -o owner [-n schema] [-U pguser] [-h host] [-p port] [-y]
常用选项：
选项	作用
-d	目标数据库名 （必填）
-o	数据库所有者用户名 （必填）
-U	连接 PostgreSQL 用的用户名（默认 peer 认证）
-h -p	主机和端口（默认 Unix socket + 5432）
-n	要创建的 schema 列表，逗号分隔
-s	只导入 schema，不导入数据
-a	只导入数据，不导入 schema
-y	自动应答 Yes，跳过所有交互确认
-f	跳过用户/数据库存在性检查，不自动创建
-x	索引和约束在数据导入之后再加
-i	只导入索引、约束和触发器
-j N	并行连接数，加速导入
-P N	同时处理的表数

是否会自动创建 PostgreSQL 中的 user / schema / table？
会，前提是没有加 -f（-f 会跳过自动创建）。
User（数据库所有者）
脚本查 pg_user，如果 $DB_OWNER 不存在，会提示并执行：
createuser --no-superuser --no-createrole --no-createdb $DB_OWNER
Database
脚本查 pg_database，如果 $DB_NAME 不存在，会提示并执行：
createdb --owner $DB_OWNER -E UTF8 $DB_NAME
如果库已存在，还会问你要不要 drop 重建。
Schema
通过 -n schema1,schema2 传入，脚本会逐条执行：
CREATE SCHEMA $lnspace;
创建完后还会把 schema 加到 owner 的 search_path 上。
Table
Ora2Pg 导出时会生成 schema/tables/table.sql，里面包含了 CREATE TABLE 等 DDL。脚本直接拿 psql -f 执行这个文件，所以表不是脚本凭空捏的，是 Ora2Pg 从 Oracle 结构转换出来的 DDL 自动创建的。
同理，View、Sequence、Trigger、Function、Procedure 等都是读取对应 SQL 文件后 psql -f 执行进去的。
总结：这个脚本是一站式导入工具 —— 从创建用户、建库、建 schema，到执行 DDL 建表、导数据、加索引约束，按提示一步步走完。加 -y 可以跳过所有交互确认，适合自动化场景。

./import_all.sh -d target_db -o postgres -n ora2pg_test -U postgres -h 172.18.0.2 -p 5432 -y
